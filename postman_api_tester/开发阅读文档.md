# 开发阅读文档（约束汇总）

版本基线：v1.0.2（2026-04-20）
适用范围：postman_api_tester 模块、report_server.py、运行与报告链路

## 1. 开发前必读条件（新增）

开始任何开发前，必须先完整阅读以下文件并确认当前改动是否触及其中约束：

1. postman_api_tester/操作手册.md
2. postman_api_tester/快速命令参考.md
3. postman_api_tester/config.py
4. report_server.py
5. postman_api_tester/postman_api_tester.py

未完成上述阅读，不允许直接修改核心逻辑（token处理、报告生成、报告服务接口、分页与筛选行为）。

## 2. 当前有效约束清单

### 2.1 运行与环境约束

1. Python 版本要求：3.10+
2. 依赖安装入口：requirements.txt
3. 建议在项目根目录执行命令：d:/tangzk/py/seldom-api-testing

### 2.2 测试执行参数约束

1. CLI 参数顺序：
   python -m postman_api_tester.postman_api_tester <postman_file> [base_url] [output_dir] [token] [results_per_page]
2. token 优先级：命令行/代码入参 token > postman_api_tester/config.py 的 TOKEN > 自动登录获取
3. base_url 优先级：命令行/代码入参 base_url > postman_api_tester/config.py 的 BASE_URL > 从 Postman 集合解析
4. output_dir 优先级：命令行/代码入参 output_dir > postman_api_tester/config.py 的 REPORT_OUTPUT_DIR > 项目根目录 reports/
5. 报告分页参数 results_per_page 默认 30（报告页建议控制在 1~100）

### 2.3 报告产物约束

每次执行必须生成同名前缀的三类文件：

1. postman_report_时间戳.html（主报告）
2. postman_report_时间戳_details.json（详情数据）
3. postman_report_时间戳_meta.json（历史索引/对比元数据）

### 2.4 报告中心约束（report_server.py）

1. 服务默认监听 0.0.0.0:5000
2. 报告目录读取优先级：
   POSTMAN_REPORTS_DIR 环境变量 > REPORTS_DIR 环境变量 > config.py 中 REPORT_OUTPUT_DIR > 项目根目录 reports/
3. 单报告数据页分页规则：
   默认 20 条/页；上限 100 条/页
4. 支持报告历史对比：新增接口、移除接口、状态变化、成功率变化
5. 并发写回同一报告时使用按报告名粒度的写锁，避免并发覆盖

### 2.5 局域网访问约束

1. 报告中心可通过 http://127.0.0.1:5000 本机访问
2. 同局域网访问需要放通 5000 端口
3. 推荐执行脚本：allow_report_server_firewall.ps1

## 3. 每次修改后更新本文件的命令（新增）

在项目根目录执行以下 PowerShell 命令，将“本次改动中疑似约束变更”的候选行自动追加到本文件末尾，作为待确认记录：

```powershell
$doc = "postman_api_tester/开发阅读文档.md"
$ts = Get-Date -Format "yyyy-MM-dd HH:mm:ss"
$pattern = "优先级|默认|必须|限制|上限|下限|分页|token|base_url|report_output_dir|POSTMAN_REPORTS_DIR|REPORTS_DIR|5000|status|meta|details"
$files = @(
  "postman_api_tester/config.py",
  "postman_api_tester/postman_api_tester.py",
  "report_server.py",
  "postman_api_tester/README.md",
  "postman_api_tester/操作手册.md",
  "postman_api_tester/快速命令参考.md",
  "postman_api_tester/最终交付清单.md"
)
"`n## 约束更新记录（$ts）`n" | Add-Content $doc
rg -n -i $pattern $files | Add-Content $doc
"`n" | Add-Content $doc
```

执行后请人工二次整理：

1. 删除误匹配行
2. 将真实变更写入“当前有效约束清单”对应章节
3. 保持优先级、默认值、边界值三类信息准确
