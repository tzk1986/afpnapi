# Postman API 测试工具 - 快速命令参考（v1.0.2）

文档统一目录：`postman_api_tester/`

## 1. 环境准备

```bash
cd d:\tangzk\py\seldom-api-testing
pip install -r requirements.txt
```

## 2. 最常用命令

```bash
# 方式1：模块方式（推荐）
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json"

# 指定 base_url
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json" "http://10.50.11.130:11000"

# 指定输出目录
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json" "http://10.50.11.130:11000" "reports"

# 指定 token（优先级最高）
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json" "http://10.50.11.130:11000" "reports" "your_token"

# 指定分页大小（第5个参数）
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json" "http://10.50.11.130:11000" "reports" "your_token" 50
```

补充：

- 如不传 `output_dir`，将按 `postman_api_tester/config.py` 中 `REPORT_OUTPUT_DIR`（为空则默认 `reports/`）

## 3. Python 调用方式

```python
from postman_api_tester.postman_api_tester import run_postman_tests

report = run_postman_tests(
    postman_file="支付（UPS）.postman.json",
    base_url="http://10.50.11.130:11000",
    output_dir="reports",
    token=None,
    results_per_page=30,
)
print(report.generate_summary())
```

## 4. 报告查看

```bash
# 生成文件在 reports/ 目录
# 例如：reports/postman_report_20260416_190000.html

# 如需历史报告、差异对比、Token测试、重新请求、局域网访问
python report_server.py
# 浏览器打开 http://localhost:5000
```

如果报告目录不是默认 `reports/`，可选两种方式：

```powershell
# 方式1：环境变量指定
$env:POSTMAN_REPORTS_DIR = "D:\api-test-reports"
python report_server.py

# 方式2：在 config.py 中设置 REPORT_OUTPUT_DIR
```

补充说明：

- 首页可查看历史报告列表
- 点击“数据视图”可进入单份报告的结果浏览页
- 单报告页支持按名称/路径搜索、状态筛选、分页和详情查看
- 可选择两份报告进行差异对比
- 同局域网可通过 `http://本机局域网IP:5000` 访问

## 5. Token 优先级

1. 命令行/代码传入 token 参数
2. postman_api_tester/config.py 的 TOKEN
3. 自动识别 login 接口并获取

## 6. 常见问题快速处理

- 报错 ModuleNotFoundError: 先执行 `pip install -r requirements.txt`
- 报错 FileNotFoundError: 检查 JSON 路径是否相对当前目录正确
- 报告打不开或功能不完整: 使用 `python report_server.py` 启动本地服务
- 控制台中文乱码: Windows 建议先执行 `$env:PYTHONIOENCODING="utf-8"`
