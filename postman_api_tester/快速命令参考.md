## 🔍 命令参数说明

```bash
python postman_api_tester.py <file> [base_url] [output_dir]

参数说明：
  <file>        必需 - Postman JSON 文件路径
  [base_url]    可选 - 覆盖文件中的 baseUrl（用于多环境测试）
  [output_dir]  可选 - 报告输出目录（默认 ../reports）

示例：
  python postman_api_tester.py ../api.json
  python postman_api_tester.py ../api.json https://test-api.example.com
  python postman_api_tester.py ../api.json https://api.example.com D:\TestReports
```

## 🐛 故障快速排查

```bash
# ❌ ModuleNotFoundError: No module named 'seldom'
# ✅ 解决：pip install -r requirements.txt

# ❌ FileNotFoundError: 文件不存在
# ✅ 解决：检查路径，使用绝对路径或相对路径正确

# ❌ SSL 连接错误
# ✅ 解决：检查网络，可重试，或禁用 SSL 验证

# ❌ 编码错误（UnicodeEncodeError）
# ✅ 解决：不使用管道，直接运行命令

# ❌ 权限错误（Permission denied）
# ✅ 解决：确保有写入 reports 目录的权限
```

## 🔐 安全提示

```bash
# ❌ 不要在命令行中硬编码密钥
python postman_api_tester.py api.json https://api?key=secret

# ✅ 改为使用环境变量
export API_KEY=secret
python postman_api_tester.py api.json https://api

# ✅ 或在 Postman 中使用变量
在 api.json 中定义变量：
  "variable": [
    { "key": "apiKey", "value": "{{API_KEY}}" }
  ]
```

## 🎁 高级用法

### 批量测试多个环境

```bash
#!/bin/bash
# 测试多个环境
for env in dev test prod; do
  python postman_api_tester.py ../api.json https://${env}-api.example.com ./reports/${env}
  echo "✓ $env 环境测试完成"
done
```

### Python 脚本集成

```python
from postman_api_tester import run_postman_tests

# 执行测试
report = run_postman_tests(
    postman_file='api.json',
    base_url='https://api.example.com',
    output_dir='./reports'
)

# 获取摘要
summary = report.generate_summary()
print(f"成功率: {summary['success_rate']}")
```
