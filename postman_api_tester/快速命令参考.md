## 3. 最常用命令

```bash
# 方式1：模块方式（推荐）
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json"

# 方式1.1：第一次打通环境建议先用示例文件
python -m postman_api_tester.postman_api_tester "sample_api_collection.json"

# 指定 base_url
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json" "http://10.50.11.130:11000"

# 指定输出目录
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json" "http://10.50.11.130:11000" "reports"

# 指定 token（优先级最高）
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json" "http://10.50.11.130:11000" "reports" "your_token"

# 指定分页大小（第5个参数）
python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json" "http://10.50.11.130:11000" "reports" "your_token" 50
```

补充：

- 如不传 `output_dir`，将按 `postman_api_tester/config.py` 中 `REPORT_OUTPUT_DIR`（为空则默认 `reports/`）
- 如不传 `token`，按当前优先级会尝试 `POSTMAN_TOKEN`、`config.py` 中的 `TOKEN`，最后再走自动登录

## 4. 配置文件最常改项

编辑文件：`postman_api_tester/config.py`

最常改的配置：

```python
BASE_URL = "http://10.50.11.130:11000"
TOKEN = ""
REPORT_OUTPUT_DIR = r"D:\api-test-reports"
REQUEST_CONNECT_TIMEOUT = 10
REQUEST_READ_TIMEOUT = 30
ENABLE_SELECTIVE_RUN = True
ENABLE_ADHOC_RUN = True
ADHOC_MAX_ITEMS = 200
ADHOC_DEFAULT_COLLECTION_NAME = "报告中心临时测试"
REPORT_EXPORT_DEFAULT_SCOPE = "full"
```

推荐顺序：

1. 先改 `BASE_URL`
2. 再决定 `TOKEN` 填不填
3. 再确认 `REPORT_OUTPUT_DIR`
4. 超时不够再改 `REQUEST_CONNECT_TIMEOUT` / `REQUEST_READ_TIMEOUT`

也可用环境变量临时覆盖：

```powershell
$env:POSTMAN_BASE_URL = "http://10.50.11.130:11000"
$env:POSTMAN_TOKEN = "your_token"
python -m postman_api_tester.postman_api_tester "sample_api_collection.json"
```

## 6. 报告查看

```bash
# 生成文件在 reports/ 目录
# 例如：reports/postman_report_20260416_190000.html

# 如需历史报告、差异对比、Token测试、重新请求、局域网访问
python report_server.py
# 浏览器打开 http://localhost:5000
```

报告中心常用入口：

- 首页：历史报告、删除报告、差异对比
- 上传区：支持先解析接口，再按“全量/仅已选接口”执行
- ad-hoc 独立页入口：支持从首页跳转到“新增接口测试”页面，再录入接口并执行生成报告（无需先准备完整 Collection 文件）
- 数据视图：查看每个接口的请求/响应详情
- 人工用例编辑器：三段式流程（发送请求 -> 回填响应 -> 选择 PASSED/FAILED 并保存）
- 编辑界面切换与 Body 类型选择：mc/er 双编辑器切换隔离，支持 none/raw/urlencoded/formdata/graphql/binary，切换时互不污染
- 人工用例编辑回显：编辑时自动回显历史请求信息（Headers/Params/Body），可改参后重发
- 人工用例详情响应体：与报告详情同款展示逻辑（预览/展开）
- 页面模板更新时间标记：报告页顶部可快速确认当前模板版本是否已更新
- 结果行内排除：在列表行直接切换“排除/取消排除”
- 原始 HTML：打开原始静态报告
- 元数据：查看结构化摘要和结果索引
- 导出最新 JSON：支持 `full` 和 `report_only`

导出范围速记（full vs report_only）：

- `full`：导出源 Collection 的全部接口（再叠加人工用例并减去已排除项）。
- `report_only`：仅导出“本次报告执行到的接口”（再叠加人工用例并减去已排除项）。
- 两者何时相同：当本次报告已覆盖源 Collection 全部接口时，导出内容会一致（页面会提示“范围等价”）。
- 选型建议：需要完整回归包选 `full`；需要最小复现包选 `report_only`。

ad-hoc 快速约束：

- 后端入口：`/api/run-ad-hoc-tests`
- 输入主体：JSON `cases` 数组，至少 1 条，数量上限受 `ADHOC_MAX_ITEMS` 控制
- URL 规则：支持绝对 `http/https`、`{{baseUrl}}/...` / `{{base_url}}/...`、或配合 `base_url` 的相对路径
- 执行链路：先写入 `uploaded_collections/<job_id>.json`，再复用既有任务状态接口与报告生成链路

人工用例发送补充：

- 发送请求走后端代理接口，仅允许 `http/https`。
- 编辑重试接口 `/re-request-api` 同步限制 `http/https`。
- 请求超时遵循 `postman_api_tester/config.py` 中 `REQUEST_CONNECT_TIMEOUT` 和 `REQUEST_READ_TIMEOUT`。

可复用冒烟脚本：

- 启动服务后执行：`python test_data/smoke_report_server_20260427.py`

如果报告目录不是默认 `reports/`，可选两种方式：

```powershell
# 方式1：环境变量指定
$env:POSTMAN_REPORTS_DIR = "D:\api-test-reports"
python report_server.py

# 方式2：在 config.py 中设置 REPORT_OUTPUT_DIR
```

补充说明：

- 首页可查看历史报告列表
- 点击“数据视图”可进入单份报告的结果浏览页
- 单报告页支持按名称/路径搜索、状态筛选、分页和详情查看
- 单报告页失败/异常结果可看到数据库迁移反馈提示
- 可选择两份报告进行差异对比

## 9. 常见问题快速处理

- 报错 ModuleNotFoundError: 先执行 `pip install -r requirements.txt`
- 报错 FileNotFoundError: 检查 JSON 路径是否相对当前目录正确
- 报告打不开或功能不完整: 使用 `python report_server.py` 启动本地服务
- 失败详情中看到“数据库反馈”: 优先按提示判断是否为连接、认证、SQL兼容、schema、字符集或类型兼容问题
- 导出后二次导入出现“预览 2 条但执行 1 条”: 当前版本已修复；执行层会在 `request.url.path` 缺失时回退解析 `request.url.raw`
- 导出后二次导入后人工用例请求体为空：当前版本已修复；导出文件现会保留人工用例完整请求参数（headers/params/body）
- “全量导出 / 导出本次报告接口”看起来一样：当前版本已补充提示；当源集合与本次报告接口集合一致时，二者导出内容本就相同，导出弹窗会说明该情况
- 导入执行接口点击“排除”不生效：当前版本已修复；排除键已统一规范并兼容旧格式，`include_excluded=false` 会正确隐藏已排除项
- 人工用例新增后仍在列表最后：当前版本已修复；旧人工用例缺失 `created_at` 时不再临时补当前时间，排序不会漂移
- 数据视图搜索不作用于人工用例列表：当前版本已修复；搜索条件已同步过滤人工用例列表
- 控制台中文乱码: Windows 建议先执行 `$env:PYTHONIOENCODING="utf-8"`
