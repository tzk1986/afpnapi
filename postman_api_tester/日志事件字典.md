# 日志事件字典（v1.3.0）

发布日期：2026-05-13
文档定位：统一列出结构化日志 `event` 命名及触发时机，便于 ELK/检索系统聚合。

## 1. 命名规则

1. 统一格式：`domain.action.state`
2. 小写英文，使用 `.` 分段，不使用空格和下划线。
3. `domain` 建议取值：`job`、`manual_case`、`report`、`handler`、`http`、`logging`、`test`。

## 2. 事件清单

### 2.1 执行链路（job/test）

- `job.run.started`：任务开始执行。
- `job.run.progress`：任务执行进度更新（采样输出）。
- `job.run.completed`：任务执行完成并已刷新报告索引。
- `job.run.failed`：任务执行失败。
- `job.enqueue.queued`：任务入队成功。
- `job.retry.queued`：重试任务入队。
- `job.retry.invalid_mode`：重试模式非法。
- `job.retry.context_failed`：重试上下文构建失败。
- `job.retry.context_ready`：重试上下文构建完成。
- `job.submit.params_built`：普通任务参数构建完成。
- `job.submit.adhoc_params_built`：ad-hoc 任务参数构建完成。
- `job.submit.path_built`：任务保存路径生成完成。
- `test.run.executed`：单接口执行结果记录。

### 2.2 人工用例链路（manual_case）

- `manual_case.add.created`：人工用例新增成功。
- `manual_case.update.updated`：人工用例更新成功。
- `manual_case.delete.deleted`：人工用例删除成功。
- `manual_case.exclusion.updated`：人工用例排除状态更新。

### 2.3 报告链路（report）

- `report.export.started`：报告导出开始。
- `report.export.completed`：报告导出完成。
- `report.delete.started`：报告删除开始。
- `report.delete.completed`：报告删除完成。

### 2.4 路由编排链路（handler）

- `handler.job.run.forward`
- `handler.job.retry.forward`
- `handler.job.enqueue.forward`
- `handler.manual_case.list.forward`
- `handler.manual_case.add.forward`
- `handler.manual_case.update.forward`
- `handler.manual_case.delete.forward`
- `handler.manual_case.exclusion.forward`
- `handler.collection.preview.extracted`
- `handler.collection.adhoc_case.normalized`
- `handler.collection.adhoc_collection.built`
- `handler.collection.selected_paths.parsed`
- `handler.http.execute.received`
- `handler.http.execute.invalid_url`
- `handler.http.execute.invalid_body`
- `handler.http.execute.completed`
- `handler.http.execute.failed`
- `handler.admin.report_delete.forward`

### 2.5 平台与访问日志（logging/http）

- `logging.configure.applied`：日志系统初始化完成。
- `http.access.logged`：服务端访问日志（请求级）。

## 3. 检索建议

1. 聚合维度优先：`event`、`status_code`、`job_id`、`report_name`。
2. 告警维度优先：`event=job.run.failed`、`event=handler.http.execute.failed`、`event=report.export.completed`（用于成功率对比）。
3. 建议按 `event` 建索引并保留原始 `logger` 字段，便于跨模块追踪。
