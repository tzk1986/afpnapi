# 竞品调研参考（GitHub 开源 API 测试工具）

## 调研日期：2026-07-02
## 调研范围：GitHub stars > 1000 的开源 API 测试/开发工具

---

## 1. Bruno — github.com/usebruno/bru（~28k stars）

**定位：** 离线优先、Git 友好的 API 客户端，集合以原生 `.bru`（Markdown 格式）文件存储于文件夹中。

**核心创新：**
- 集合直接以文本文件存储在本地目录，天然支持 git diff/branch/merge，无需导入/导出
- 无云端依赖，完全离线工作
- JS Pre-request 脚本（与本项目 v1.18.0 功能对标）
- 插件架构支持扩展

**可借鉴点：**
- **Git 友好集合存储**：本项目当前以 JSON 格式存储集合，可考虑增加"导出为文本格式"选项，便于版本控制
- **文件夹即目录**：Bruno 的文件夹嵌套直接映射到文件系统目录，比 Postman 的嵌套 ID 方案更简洁
- **本地优先定位**：Bruno 因 Postman 云端化转型而快速崛起，证明"本地优先、Git 友好"是强市场需求

---

## 2. Hoppscotch — github.com/hoppscotch/hoppscotch（~68k stars）

**定位：** 完全开源（MIT），轻量级 API 开发生态，Vue.js/Nuxt 3 + TypeScript 构建。

**核心创新：**
- 真正的 PWA：支持离线使用，可安装为桌面/移动应用
- 实时协作：共享工作区 + 实时会话
- 多协议支持：REST + WebSocket + Socket.IO + GraphQL
- 丰富的键盘快捷键（Ctrl+Enter 发送、Ctrl+K 命令面板）
- 分屏编辑器（左侧请求参数、右侧/底部响应）

**可借鉴点：**
- **PWA 能力**：本项目当前使用 Flask 模板渲染，可考虑增加 Service Worker 支持，实现离线缓存
- **键盘快捷键体系**：当前 Collection 编辑器缺少快捷键支持，可参考 Hoppscotch 的快捷键设计
- **分屏布局**：请求/响应并排展示，减少页面滚动
- **暗色主题优先**：开发者工具默认暗色主题更符合使用习惯

---

## 3. Schemathesis — github.com/schemathesis/schemathesis（~6k stars）

**定位：** 基于属性的 API 测试框架，从 OpenAPI/Swagger 规范自动生成数千个测试用例。

**核心创新：**
- 从 OpenAPI 规范自动生成测试 payload，无需手动编写测试用例
- 集成 Hypothesis（Python 主流属性测试库）发现边界情况
- 原生 pytest 插件
- 支持 OpenAPI 2.0/3.0 和 GraphQL

**可借鉴点：**
- **自动断言生成**：如果用户提供 OpenAPI 规范，可从 Schema 自动生成基线断言（必填字段、类型、枚举值）
- **与现有断言引擎结合**：本项目已有 `assertions.py`（JSONPath 断言），可增加"从 OpenAPI 导入断言规则"功能
- **减少手动配置**：用户只需提供规范文件，测试自动生成

---

## 4. Tavern — github.com/taverntesting/tavern（~2k stars）

**定位：** YAML 格式的 API 测试框架，与 pytest 集成。

**核心创新：**
- 纯 YAML 编写测试，无需 Python 代码
- 变量在测试步骤间插值（请求串联）
- 在 pytest 内运行，享受 pytest fixture/参数化/报告

**可借鉴点：**
- **YAML 输入格式**：本项目当前只支持 Postman JSON，可增加 YAML 格式作为轻量级替代
- **变量链式传递**：Tavern 的变量提取/注入语法更简洁，可参考优化本项目的 `x_extract` 语法

---

## 5. HTTPie — github.com/httpie/httpie（~30k stars）

**定位：** 现代化 CLI HTTP 客户端，语法简洁、输出美观。

**核心创新：**
- 自然语言式请求语法：`http POST example.com name=user`
- 响应体按内容类型自动语法高亮
- 代码片段生成（curl、Python requests、JavaScript fetch）
- 命令历史 + 模糊搜索

**可借鉴点：**
- **代码片段生成**：本项目已有此功能，HTTPie 证明这是高频需求
- **自然语法减少认知负担**：请求构建界面可参考 HTTPie 的"自然语言"设计理念

## 优先级排序（对本项目的适用性）

| 优先级 | 借鉴方向 | 对应竞品 | 实现难度 |
|--------|---------|---------|---------|
| **P0** | Collection 编辑器快捷键体系 | Hoppscotch | 低（纯前端） |
| **P1** | 自动断言生成（从 OpenAPI） | Schemathesis | 中（需解析规范） |
| **P2** | Git 友好集合存储格式 | Bruno | 中（需新格式） |
| **P3** | PWA 离线缓存 | Hoppscotch | 高（架构调整） |
| **P4** | YAML 输入格式支持 | Tavern | 低（新增解析器） |
