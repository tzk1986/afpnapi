## 1. 交付目标

完成接口测试工具、报告中心、交付文档的统一整理，保证新人拿到代码或压缩包后，能按统一说明完成安装、配置、执行测试与查看报告。

### 2.2 当前能力已包含

1. 命令行和交互式两种执行方式。
2. 报告三件套输出：`.html`、`_details.json`、`_meta.json`。
3. 报告中心历史列表、数据视图、删除报告、差异对比。
4. 报告目录可由环境变量或配置文件统一控制。
5. 数据库迁移相关异常识别与反馈建议。
6. 新人入口文档、完整手册、命令速查三层文档结构。
7. 上传后支持“全量执行 / 仅执行已选接口”。
8. 导出支持 `full`（全量）与 `report_only`（本次报告接口）双模式。
9. 报告页支持人工用例三段式流程（发送请求 -> 回填响应 -> 选择 PASSED/FAILED -> 保存）。
10. 结果列表支持行内“排除/取消排除”，减少详情弹窗操作路径。
11. 人工用例发送请求由后端代理接口统一处理，限制 `http/https` 协议并复用配置化超时。
12. 报告页新增页面模板更新时间小标记，支持快速确认页面是否为最新模板。
13. 人工用例详情响应体展示与报告详情统一，支持预览与展开查看。
14. 编辑人工用例支持回显历史请求信息（Headers/Params/Body），可改参后重新请求。
15. 导出一致性修复：导出弹窗人工用例数量与实际导出一致，且已排除接口不会进入导出文件。
16. 修复报告页双编辑器 body 模式切换互相污染问题（mc/er 隔离）。
17. 编辑界面切换与 Body 类型选择体验优化：支持 none/raw/urlencoded/formdata/graphql/binary，且与重试区切换互不干扰。
18. `/re-request-api` 与 `/api/proxy-request` 均限制为 `http/https`，统一 SSRF 防护。
19. 请求体构建逻辑统一，减少两路由重复代码并保持行为一致。
20. 提供可复用运行时冒烟脚本：`test_data/smoke_report_server_20260427.py`。
21. 导入执行一致性修复：导出后二次导入时，`request.url` 仅含 `raw`（无 `path`）的接口不再被跳过，避免“解析 2 条、执行 1 条”。
22. 导出参数完整性修复：导出文件中的人工用例保留完整请求参数（headers/params/body/body_mode/body_data），导入后可完整解析并执行。
23. 报告中心首页新增 ad-hoc 直接执行能力：可不上传集合文件，直接录入接口并生成标准报告三件套。
24. ad-hoc 直接执行继续复用既有任务队列、状态轮询与报告查看链路，临时 Collection 文件写入 `uploaded_collections/`。
25. 人工用例排序稳定性修复：新增人工用例在列表中按最新优先展示，且旧数据缺失 `created_at` 不再导致排序漂移。
26. 数据视图搜索一致性修复：搜索词、状态筛选与“显示已排除”开关同步作用于人工用例列表。
27. 排除链路一致性修复：报告结果与人工用例排除键统一规范并兼容旧格式，导入执行接口的排除/隐藏/导出剔除链路一致生效。
28. 导出范围可解释性增强：当 `report_only` 与 `full` 实际导出内容一致时，导出返回与前端弹窗会明确提示“当前范围等价”。
29. 导出范围使用口径明确化：文档明确区分 `full`（完整回归包）与 `report_only`（最小复现包），并说明两者“何时不同、何时相同、如何选择”。
30. GET 参数重复修复：执行层、重试接口与代理接口统一采用 URL+params 去重合并策略，避免 query 重复发送。
31. 详情人工判定能力：结果详情新增“标记成功/标记失败/恢复自动结果”，并回写 `meta` 结果与 summary 统计。
32. **响应时间记录与展示**：每接口记录 `response_time_ms`，数据视图结果列表新增"耗时(ms)"列，底部汇总区显示 avg/max/p95 响应时间；受 `ENABLE_RESPONSE_TIME` 控制。
33. **一键重试失败用例**：报告页工具栏"重试失败用例"按钮，后端 `POST /api/retry-failures` 自动重跑 FAILED/ERROR 接口并生成新报告；受 `ENABLE_RETRY_FAILURES` 控制。
34. **多环境配置切换**：首页上传表单与 ad-hoc 页面支持"环境"下拉框，选择后 `base_url`/`token` 跟随切换；通过 `ENVIRONMENTS` 配置或 `ENVIRONMENTS_JSON` 环境变量注入；`GET /api/environments` 接口提供环境列表。
35. **JSONPath 断言规则**：ad-hoc 页面（`ENABLE_ASSERTIONS=true` 时显示）支持填写 JSON 格式断言，执行时由 `assertions.py` 对响应体进行 JSONPath 校验，支持 eq/ne/gt/lt/gte/lte/contains/exists/not_exists；需要依赖 `jsonpath-ng>=1.5.3`。
36. **JUnit XML 导出**：报告页工具栏"导出 JUnit XML"链接，`GET /api/export-junit/<name>` 生成标准 JUnit XML 报告供 CI/CD 管线（Jenkins/GitLab CI 等）消费；受 `ENABLE_JUNIT_EXPORT` 控制；使用 stdlib `xml.etree.ElementTree`，无额外依赖。
37. **首页列表筛选增强**：首页报告列表支持通过状态（全部/仅通过/仅失败/仅错误）、最低通过率、日期范围筛选，一键重置；筛选逻辑在前端 `renderReports()` 中实现；受 `ENABLE_REPORT_LIST_FILTER` 控制。
38. **断点恢复（v1.2.1）**：执行链路支持 checkpoint（按 `item_path`）恢复执行，默认关闭，开启后可跳过已执行接口。
39. **部分成功报告（v1.2.1）**：执行中断时仍输出报告三件套，并在 `_meta.json` 写入 `execution_mode/interrupted/interrupt_reason`。
40. **断言严格模式（v1.2.1）**：新增 `ENABLE_ASSERTION_STRICT_MODE`，控制断言引擎异常是否严格判定为 FAILED。
41. **报告列表摘要懒加载（v1.2.1）**：首页列表仅读取 meta 摘要字段，详情按需加载完整 `results`。
42. **流式导出（v1.2.1）**：新增 `POST /api/export-collection-stream`，旧接口 `/api/export-collection` 保持兼容。
43. **结果分块渲染（v1.2.1）**：数据视图结果表支持前端分块绘制，降低大页渲染阻塞。
44. **导出通道三态（v1.2.1）**：新增 `REPORT_EXPORT_CHANNEL_MODE`（`auto` / `legacy` / `stream`）与 `REPORT_EXPORT_STREAM_THRESHOLD`，支持自动分流与排障强制通道。
45. **分析洞察（2.1）**：单报告页新增 analytics 页签，支持状态/方法/目录分布、响应时间直方图、错误分类、建议、质量评分、覆盖率与趋势视图；接口入口为 `GET /api/report-analytics/<report_name>`。
46. **分析对比（2.1）**：analytics 页签支持左右报告对比，返回 `left_snapshot/right_snapshot/delta/score_delta`；接口入口为 `GET /api/report-analytics-compare`。
47. **分析洞察图表可视化（2.1 图表增强）**：analytics 页签新增 Chart.js 六大图表（状态分布环形图、方法分布环形图、响应时间直方图、错误分类柱状图、质量评分雷达图、趋势双折线图），与现有表格数据并排共存；支持 CDN 失败自动降级为纯表格；受 `ENABLE_REPORT_ANALYTICS_CHARTS` 控制，且受父开关 `ENABLE_REPORT_ANALYTICS` 联动约束。
48. **可配置结果自动判定（v1.3.3）**：支持通过 errCode 和 message 可配置条件自动判断测试结果（PASSED/FAILED），消除手动标记。四层优先级覆盖：Web 任务级 > 集合接口级 x_* 字段 > 全局 config.py > 内置默认值。首页与 ad-hoc 页面提供"填写即启用"文本输入框（placeholder 显示默认值），不填写则使用全局配置；集合内 `request` 对象支持 `x_success_err_codes`/`x_success_messages`/`x_enable_err_code_judgment`/`x_enable_message_judgment` 接口级覆盖。不配置时行为与 v1.3.2 完全一致。
49. **数据驱动测试（v1.4.0）**：支持 CSV/JSON 外部数据文件，每行/每对象作为一组测试数据替换集合中的 `{{变量}}`。仅引用数据变量的接口按行数展开，无变量引用的接口保持单次执行。`ENABLE_DATA_DRIVEN` 默认关闭，`DATA_FILE_MAX_ROWS` 默认 10000。Web UI 首页上传区条件显示数据文件上传入口。报告列表在接口名称后显示数据组编号徽标。checkpoint 支持 `item_path#data_index` 复合键。
50. **请求串联与变量提取（v1.4.0）**：支持在 Postman Collection `request` 对象中配置 `x_extract` 字段，使用 JSONPath 表达式（`$.field`、`$.array[0].id`）或 `$header.Name` 前缀从响应中提取变量。提取的变量供后续接口的 `{{变量名}}` 引用。`ENABLE_VARIABLE_EXTRACTION` 默认关闭。Web UI 首页上传区条件显示预置变量编辑区。详情面板显示提取的变量键值对。ad-hoc 页面支持配置 `x_extract`。
51. **Collection 可视化编辑器（v1.5.0）**：导入 Postman Collection v2.1 JSON 后在线编辑接口参数（URL/Method/Headers/Params/Body/x_extract）。树形导航 + 分组管理。首页新增"可视化编辑 Collection"入口卡片。访问路径 `/collection-editor`。
52. **单接口发送与调试（v1.5.0）**：编辑器内直接 Send 单个请求，查看响应状态码/耗时/Headers/Body。响应 Body 以 JSON 树渲染，每个字段旁有 Extract 按钮，点击自动写入该接口的 x_extract 配置。
53. **运行时变量池（v1.5.0）**：Send 后自动执行该接口的 x_extract 规则，提取值存入运行时变量池。后续 Send 时自动将预置变量和运行时变量合并后替换 `{{变量}}`。支持手动清除运行时变量。
54. **批量选择删除（v1.5.0）**：编辑器中每个接口旁有复选框，支持全选/批量删除。方便导入后清理不需要的接口再导出精简版 Collection。
55. **变量依赖分析增强（v1.5.0）**：依赖分析结果以表格形式展示，显示变量的生产接口→消费接口流向关系，可点击跳转到对应接口的编辑面板。
56. **统一错误码体系（v1.5.0）**：80 处 API 错误全部编码，格式为 `模块前缀_序号`（`CE_/COL_/JOB_/RPT_/HTTP_/COM_`）。`base_handler.json_error()` 增加 error_code 参数。前端错误提示显示 `[错误码] 提示信息` 格式。网络错误 fallback 为 `COM_NET_001`。操作手册新增第 14 节"错误码速查表"。
57. **页内执行结果面板（v1.6.0）**：可视化编辑器点击"执行"后弹出模态框，实时轮询显示执行进度（百分比/完成数/总数），完成后展示结果摘要与查看报告链接。关闭弹窗后编辑器状态完整保留，不再跳转页面丢失编辑内容。
58. **接口拖拽排序（v1.6.0）**：可视化编辑器左侧接口列表支持 HTML5 拖拽调整顺序。拖拽后自动更新 `state.collection.groups` 中的顺序，导出和执行均按页面显示顺序进行。
59. **Body 变量替换修复（v1.6.0）**：当 raw body 包含 `{{variable}}` 占位符导致 `json.loads()` 失败时，body 保留为字符串。变量替换后若 body 以 `{` 或 `[` 开头，自动重新解析为 JSON 对象，避免 requests 库将字符串序列化为 JSON 字符串导致服务器返回 400。
60. **运行时变量池文档（v1.6.0）**：操作手册新增 5.5 节，以实际示例说明变量池工作原理、使用场景和清除时机。
61. **变量自动补全（v1.7.0）**：编辑器中输入 `{{` 自动弹出可用变量列表（预置变量/运行时变量/x_extract 提取规则），支持键盘导航（↑/↓/Enter/Escape）、实时过滤、点击或回车插入，自动闭合 `}}`。来源标签区分"预置"、"运行时"、"提取规则"三类。
62. **空白接口新增（v1.8.0）**：侧边栏"+ 新增"下拉菜单 → "空白接口"，填写名称/方法/URL/描述并选择目标分组后创建。空 Collection 时自动初始化。
63. **接口复制（v1.8.0）**：支持复制当前接口或批量复制已选接口到任意分组。右键接口可"复制接口"（同分组）或"复制到..."（弹出分组选择器）。副本名称自动追加 `_副本`/`_副本2`/...，所有字段深拷贝。
64. **cURL 导入（v1.8.0）**：粘贴 cURL 命令解析生成接口，支持 20+ 参数（`-X`/`-H`/`-d`/`-F`/`-G`/`-u`/`-A`/`-e`/`-b`/`--proxy`/`--cert`/`-k`/`-L` 等），自动推断 body_mode，解析预览后确认导入。
65. **HAR 导入（v1.8.0）**：粘贴或上传 HAR 文件，批量选择要导入的接口。可选从响应 body 自动提取顶层 JSON 字段到 x_extract。自动过滤浏览器内部 header（Host/Connection/HTTP2 伪 header）。
66. **Postman 代码片段导入（v1.8.0）**：支持 Python(requests)/JS(fetch/axios)/Go(net/http)/Java(OkHttp)/cURL 代码片段。自动检测语言，正则提取 URL/Method/Headers/Body。
67. **新建文件夹（v1.8.0）**：创建多层嵌套文件夹，无限嵌套深度。文件夹标题可双击重命名，标题旁显示拖拽手柄和删除/重命名按钮。删除非空文件夹时接口自动移到父级分组。
68. **文件夹拖拽移动（v1.8.0）**：拖拽文件夹到另一文件夹成为子文件夹，拖到根目录区域移出到顶层。循环引用检测（不允许拖入自己的子文件夹）。
69. **接口跨文件夹拖拽（v1.8.0）**：接口可拖拽到任意文件夹标题上移入该文件夹。勾选多个接口后拖拽可批量移动。拖拽到根目录区域可移回根分组。
70. **Collection Editor 路由层测试（v1.8.1）**：新增 `test_collection_editor_routes.py`，19 个测试用例覆盖 4 个路由的全部成功/错误路径，覆盖 12 个错误码（CE_PARSE/SAVE/DEP/SEND）。
71. **Collection 路由层测试（v1.8.2）**：新增 `test_collection_routes.py`，12 个测试用例覆盖 COL_PREVIEW_001-004、COL_EXPORT_001-003 错误码。
72. **Report Result 路由层测试（v1.8.2）**：新增 `test_report_result_routes.py`，15 个测试用例覆盖 5 个路由函数的成功/错误路径。
73. **Security 工具函数测试（v1.8.2）**：新增 `test_security.py`，16 个测试用例覆盖 sanitize_headers、strip_sensitive_headers、strip_auth_headers。
74. **Report JUnit Service 测试（v1.8.2）**：新增 `test_report_junit_service.py`，15 个测试用例覆盖 JUnit XML 生成。

## 5. 本次交付的关键配置点

交付时至少需要提醒对方以下配置：

1. `BASE_URL`：目标系统地址。
2. `TOKEN`：认证 token，可为空。
3. `REPORT_OUTPUT_DIR`：报告输出目录。
4. `REQUEST_CONNECT_TIMEOUT` / `REQUEST_READ_TIMEOUT`：请求超时。
5. `RUN_JOBS_MAX`：报告中心任务记录上限。
6. `ENABLE_SELECTIVE_RUN`：是否启用仅执行已选接口。
7. `ENABLE_ADHOC_RUN`：是否启用首页“新增接口测试”独立页面入口。
8. `ADHOC_MAX_ITEMS` / `ADHOC_DEFAULT_COLLECTION_NAME`：ad-hoc 单次任务上限与默认集合名。
9. `REPORT_EXPORT_DEFAULT_SCOPE`：导出默认范围（`full` / `report_only`）。
10. `REPORT_EXPORT_CHANNEL_MODE`：导出通道模式（`auto` / `legacy` / `stream`）。
11. `REPORT_EXPORT_STREAM_THRESHOLD`：自动分流阈值（总接口数 >= 阈值时优先走流式）。
12. `ENABLE_RESPONSE_TIME`：是否记录并展示响应时间（默认 `True`）。
13. `ENABLE_RETRY_FAILURES`：是否启用报告页一键重试失败用例按钮（默认 `True`）。
14. `ENABLE_JUNIT_EXPORT`：是否启用 JUnit XML 导出链接（默认 `True`）。
15. `ENABLE_REPORT_LIST_FILTER`：是否启用首页报告列表筛选面板（默认 `True`）。
16. `ENABLE_REPORT_ANALYTICS`：是否启用 analytics 页签及接口（默认 `True`）。
16.1. `ENABLE_REPORT_ANALYTICS_CHARTS`：是否启用 analytics 图表可视化（默认 `True`）；父开关 `ENABLE_REPORT_ANALYTICS=false` 时，无论子开关为何值，页面均不加载 Chart.js、不渲染图表。
17. `REPORT_ANALYTICS_TOP_N_DEFAULT` / `REPORT_ANALYTICS_TOP_N_MAX`：analytics TopN 默认值与上限。
18. `REPORT_ANALYTICS_TREND_LIMIT_DEFAULT` / `REPORT_ANALYTICS_TREND_LIMIT_MAX`：趋势默认值与上限。
19. `REPORT_ANALYTICS_ENABLE_SAMPLES`：是否默认返回错误样本（默认 `False`）。
20. `REPORT_ANALYTICS_HISTOGRAM_BUCKETS`：响应时间桶配置。
21. `QUALITY_SCORE_FAILED_PENALTY` / `QUALITY_SCORE_ERROR_PENALTY` / `QUALITY_SCORE_SLOW_PENALTY` / `QUALITY_SCORE_ASSERTION_MISSING_PENALTY`：质量评分扣分阈值。
22. `ENABLE_ASSERTIONS`：是否启用 JSONPath 断言校验（默认 `False`，需显式开启后 ad-hoc 断言功能才可见）。
23. `ASSERTIONS_ENGINE`：断言引擎类型（默认 `jsonpath`）。
24. `ENVIRONMENTS`：多环境配置字典，或通过 `ENVIRONMENTS_JSON` 环境变量注入 JSON 字符串。
25. `DEFAULT_ENV_NAME`：默认激活的环境名称。
26. `ENABLE_CHECKPOINT_RECOVERY`：是否启用断点恢复（默认 `False`）。
27. `CHECKPOINT_FLUSH_EVERY_N`：checkpoint 写入频率（默认 `1`）。
28. `CHECKPOINT_DIR`：checkpoint 目录（空则 `reports/checkpoints`）。
29. `ENABLE_ASSERTION_STRICT_MODE`：断言引擎异常严格失败开关（默认 `False`）。
30. `LOG_LEVEL` / `LOG_FORMAT` / `LOG_SAMPLE_RATE`：日志级别、格式、采样率。
31. `LOG_ALERT_ERROR_WINDOW_SECONDS` / `LOG_ALERT_ERROR_RATE_THRESHOLD_PER_MIN`：ERROR 速率告警窗口与阈值。
32. `ENABLE_CONCURRENT`：是否启用并发执行引擎（默认 `false`）。启用后接口按变量依赖拓扑排序并行执行。
33. `CONCURRENT_WORKERS`：并发执行最大线程数（默认 `10`），仅 `ENABLE_CONCURRENT=true` 时生效。
34. `ENABLE_VARIABLE_FUNCTIONS`：是否启用内置变量函数（`{{timestamp()}}` 等 6 个，默认 `true`）。关闭后函数表达式原样保留。
35. `GLOBAL_VARIABLES_FILE`：全局变量持久化文件路径（默认空，不启用）。设置后执行前加载、执行后保存。
36. `GLOBAL_VARIABLES_MAX_COUNT`：全局变量最大数量（默认 `1000`），超出截断。
37. `ENABLE_ERR_CODE_JUDGMENT`：是否启用 errCode 参与结果判定（默认 `False`，保持旧行为）。
38. `SUCCESS_ERR_CODES`：视为成功的 errCode 值列表，逗号分隔（默认 `"0"`）。
39. `ENABLE_MESSAGE_JUDGMENT`：是否启用 message 参与结果判定（默认 `True`）。
40. `SUCCESS_MESSAGES`：视为成功的 message 值列表，逗号分隔（默认 `"success"`）。
如不希望对方改代码文件，也可以提示使用环境变量：

$env:POSTMAN_BASE_URL = "http://10.50.11.130:11000"
$env:POSTMAN_TOKEN = "your_token"
python -m postman_api_tester.postman_api_tester "sample_api_collection.json"
