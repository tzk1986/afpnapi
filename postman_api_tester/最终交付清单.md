完成接口测试工具、报告中心、交付文档的统一整理，保证新人拿到代码或压缩包后，能按统一说明完成安装、配置、执行测试与查看报告。

## 2. 本次交付范围

### 2.1 保留并交付的核心内容

- 核心执行模块：`postman_api_tester/__init__.py`、`postman_api_tester/postman_api_tester.py`
- 配置文件：`postman_api_tester/config.py`

- 报告目录：`reports/`

- 防火墙脚本：`allow_report_server_firewall.ps1`
- 依赖文件：`requirements.txt`

### 2.2 当前能力已包含

1. 命令行和交互式两种执行方式。
2. 报告三件套输出：`.html`、`_details.json`、`_meta.json`。
3. 报告中心历史列表、数据视图、删除报告、差异对比。
4. 报告目录可由环境变量或配置文件统一控制。
5. 数据库迁移相关异常识别与反馈建议。
6. 新人入口文档、完整手册、命令速查三层文档结构。
7. 上传后支持“全量执行 / 仅执行已选接口”。
8. 导出支持 `full`（全量）与 `report_only`（本次报告接口）双模式。
9. 报告页支持人工用例三段式流程（发送请求 -> 回填响应 -> 选择 PASSED/FAILED -> 保存）。
10. 结果列表支持行内“排除/取消排除”，减少详情弹窗操作路径。
11. 人工用例发送请求由后端代理接口统一处理，限制 `http/https` 协议并复用配置化超时。
12. 报告页新增页面模板更新时间小标记，支持快速确认页面是否为最新模板。
13. 人工用例详情响应体展示与报告详情统一，支持预览与展开查看。
14. 编辑人工用例支持回显历史请求信息（Headers/Params/Body），可改参后重新请求。
15. 导出一致性修复：导出弹窗人工用例数量与实际导出一致，且已排除接口不会进入导出文件。
16. 修复报告页双编辑器 body 模式切换互相污染问题（mc/er 隔离）。
17. 编辑界面切换与 Body 类型选择体验优化：支持 none/raw/urlencoded/formdata/graphql/binary，且与重试区切换互不干扰。
18. `/re-request-api` 与 `/api/proxy-request` 均限制为 `http/https`，统一 SSRF 防护。
19. 请求体构建逻辑统一，减少两路由重复代码并保持行为一致。
20. 提供可复用运行时冒烟脚本：`test_data/smoke_report_server_20260427.py`。
21. 导入执行一致性修复：导出后二次导入时，`request.url` 仅含 `raw`（无 `path`）的接口不再被跳过，避免“解析 2 条、执行 1 条”。
22. 导出参数完整性修复：导出文件中的人工用例保留完整请求参数（headers/params/body/body_mode/body_data），导入后可完整解析并执行。

## 4. 交付后对方应按什么顺序使用

建议告知对方按以下顺序执行：

1. 先阅读 `postman_api_tester/README.md` 了解目录和上手顺序。
2. 安装 Python 3.10+。
3. 在项目根目录执行 `pip install -r requirements.txt`。
4. 修改 `postman_api_tester/config.py` 中的 `BASE_URL`、`TOKEN`、`REPORT_OUTPUT_DIR`。
5. 先用 `sample_api_collection.json` 或最小 Collection 验证环境打通。
6. 再执行正式业务 `*.postman.json`。
7. 通过 `python report_server.py` 启动报告中心。
8. 进入 `http://127.0.0.1:5000` 查看历史报告、失败详情和数据库反馈。

## 5. 本次交付的关键配置点

交付时至少需要提醒对方以下配置：

1. `BASE_URL`：目标系统地址。
2. `TOKEN`：认证 token，可为空。
3. `REPORT_OUTPUT_DIR`：报告输出目录。
4. `REQUEST_CONNECT_TIMEOUT` / `REQUEST_READ_TIMEOUT`：请求超时。
5. `RUN_JOBS_MAX`：报告中心任务记录上限。
6. `ENABLE_SELECTIVE_RUN`：是否启用仅执行已选接口。
7. `REPORT_EXPORT_DEFAULT_SCOPE`：导出默认范围（`full` / `report_only`）。

如不希望对方改代码文件，也可以提示使用环境变量：

```powershell
$env:POSTMAN_BASE_URL = "http://10.50.11.130:11000"
$env:POSTMAN_TOKEN = "your_token"
python -m postman_api_tester.postman_api_tester "sample_api_collection.json"
```
