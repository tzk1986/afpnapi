范围：2.1 测试结果分析与报告洞察收尾验收

## 1. 已完成项

1. 后端 analytics 聚合能力已实现，覆盖分布、性能、错误诊断、质量评分、覆盖率与趋势。
2. 已新增接口：`GET /api/report-analytics/<report_name>`、`GET /api/report-analytics-compare`。
3. 前端 `report_view.html` 已新增“分析洞察”页签，并支持左右报告对比。
4. 已新增 2 个可复用 smoke 脚本：
   - `python -m test_data.smoke_report_analytics_20260514`
   - `python -m test_data.smoke_report_analytics_compare_20260514`
5. 四份关联文档与开发阅读文档已同步 2.1 配置项、接口、使用方式与验证口径。

## 2. 未完成项

1. 未单独沉淀 analytics JSON Schema 文件；当前以实现与 smoke 结构断言作为契约基线。
2. 未建立独立 analytics 短 TTL 缓存命名空间；当前仍依赖现有报告读取链路与参数限流控制耗时。
3. 未形成单独的页面性能基线报告与大报告内存对比记录。

## 3. 风险与遗留

1. 当报告规模继续增大时，analytics 聚合仍可能成为读热点，后续可补独立缓存与性能采样。
2. 错误分类规则当前为规则字典驱动，存在误判可能；未命中项已统一归入 `unknown`。
3. 讨论稿中的“首发默认关闭开关”未固化为默认值；如需灰度，需由部署环境显式设置 `ENABLE_REPORT_ANALYTICS=false`。

## 4. 回滚验证

1. 设置环境变量 `ENABLE_REPORT_ANALYTICS=false` 后启动服务，页面不显示 analytics 页签。
2. `GET /api/report-analytics/<report_name>` 与 `GET /api/report-analytics-compare` 在关闭开关后应返回 403。
3. 其余既有页面、报告结果、人工用例、导出与历史对比链路不依赖 analytics，可保持原行为。
