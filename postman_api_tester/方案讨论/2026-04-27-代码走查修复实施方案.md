## 3. 优先级处理清单与实施方案

### P0-1（B1） report_view.html: switchManualBodyMode 全局污染 er 编辑器

问题位置：
- templates/report_view.html（函数 switchManualBodyMode，约 L584）

现象与风险：
- 当前使用 document.querySelectorAll('.body-mode-btn' / '.body-panel') 全局切换。
- er 编辑器节点同时包含 .body-mode-btn/.body-panel，导致切 mc 时会污染 er 视图状态。

实施方案（推荐）
1. 将 switchManualBodyMode 的 DOM 查询限定到 mc 容器内（建议容器：#mc-body-editor）。
2. 仅切换 mc 范围下的 .body-mode-btn 与 .body-panel。
3. 不调整 er 现有 class，最小变更降低风险。

备选方案：
- 从 er 节点移除共享类 .body-mode-btn/.body-panel，仅保留 .er-*；但会影响现有样式复用，风险高于范围限定方案。

验证点：
1. 在 mc 中切换 none/formdata/urlencoded/raw/binary/graphql，不影响 er 当前模式。
2. 在 er 中切换模式，不影响 mc 当前模式。
3. 打开编辑弹窗后多次切换模式，按钮 active 与面板显示始终一致。

### P1-3（F1） report_server.py: _set_request_body 不支持 body_mode 六模式

问题位置：
- report_server.py（_set_request_body，约 L1087）

现象与风险：
- 当前仅支持 dict/list 或 string 转 raw。
- 对 formdata/urlencoded/graphql/binary/none 缺乏结构化回填，影响导出一致性与重试保存准确性。

实施方案：
1. 扩展 _set_request_body 签名，支持：
   - body_mode（none/raw/urlencoded/graphql/formdata/binary/legacy）
   - body_data（结构化数据）
   - legacy_body（兼容旧字段）
2. 统一模式映射规则：
   - none: 删除 body
   - raw: 写 mode=raw + raw 文本 + language
   - urlencoded: 写 mode=urlencoded + 数组项
   - formdata: 写 mode=formdata + text/file 项（文件项仅保留 file_name/src 元信息，不写本地绝对路径）
   - graphql: 写 mode=graphql + query/variables
   - binary: 写 mode=file + src（若不可用则仅保留占位）
   - legacy: 沿用旧逻辑
3. 调用点改为显式传参，避免隐式分支散落。

验证点：
1. 六模式分别保存后导出，检查 collection 请求体结构正确。
2. 旧数据（仅 body 字段）导出仍兼容。
3. 前端编辑后重试保存，再次打开详情可正确回显。
