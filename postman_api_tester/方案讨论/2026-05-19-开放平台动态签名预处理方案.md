# 开放平台动态签名预处理方案（Web 配置版）

版本：方案讨论版 v2
生成日期：2026-05-19
文档定位：解决从 APIFox 导出 Postman JSON 后执行报错"v1版本标准开放-时间戳异常(超时或错误)"的问题。在报告中心 Web 界面提供签名配置管理能力，支持多签名规则并存与复用。

---

## 一、问题现象与根因分析

### 1.1 现象描述

用户从 APIFox 导出 Postman Collection JSON 文件后，在本项目执行测试，接口返回：

```json
{
  "message": "v1版本标准开放-时间戳异常(超时或错误)"
}
```

### 1.2 根因分析

该报错是典型的**开放平台 API 签名时间戳校验失败**，核心原因如下：

| 环节 | 问题 |
|------|------|
| APIFox 调试时 | 时间戳和签名由 APIFox 的**前置脚本**或**动态变量**实时生成，每次请求都是最新的 |
| 导出 Postman JSON 时 | 时间戳和签名被**静态固化**到 JSON 文件中，成为固定值 |
| 本项目执行时 | 直接读取 JSON 中的静态参数发送请求，时间戳已过期 |
| 服务端校验时 | 发现时间戳超出有效窗口（通常 ±30s~300s），返回时间戳异常 |

**本质**：本项目当前仅支持"静态请求配置 -> 直接发送"的执行模式，缺少"请求发送前动态预处理"的能力。

### 1.3 开放平台签名机制通用特征

根据行业通用实践，开放平台签名通常包含以下要素：

1. **时间戳参数**：`timestamp`，单位通常为秒(s)或毫秒(ms)
2. **应用标识**：`app_key` / `app_id` / `access_key`
3. **签名参数**：`sign` / `signature`
4. **签名算法**：MD5 / SHA256 / HMAC-SHA256 等
5. **签名字段排序**：按参数名 ASCII 升序排序后拼接
6. **密钥拼接**：末尾拼接 `app_secret`
7. **可选随机数**：`nonce` 防重放攻击

## 三、方案设计（Web 配置版）

### 3.1 总体思路

新增**签名配置管理中心**，在报告中心 Web 界面提供独立的签名规则管理页面：

1. **Web 配置层**：用户可在浏览器中新增/编辑/删除/启用多套签名配置
2. **持久化层**：签名配置以 JSON 文件形式持久化，原子读写
3. **执行预处理层**：`PostmanTestExecutor.execute_test()` 发送请求前，读取所有启用的签名配置，按 URL 匹配规则找到适用的配置，动态更新时间戳和重新计算签名
4. **复用能力**：不同 Collection、不同接口可共享同一套签名配置，也可各自使用独立配置

### 3.2 数据模型设计

#### SignatureConfig（签名配置对象）

```python
from typing import Any, Dict, List, Optional, TypedDict


class SignatureConfig(TypedDict):
    """签名配置数据模型。"""

    id: str                           # UUID，唯一标识
    name: str                         # 配置名称（如"XX开放平台-生产环境"）
    enabled: bool                     # 是否启用
    app_key: str                      # 应用标识值
    app_secret: str                   # 应用密钥（敏感）
    app_key_param: str                # 应用标识参数名（默认 app_key）
    timestamp_param: str              # 时间戳参数名（默认 timestamp）
    sign_param: str                   # 签名参数名（默认 sign）
    sign_algorithm: str               # 签名算法：md5 / sha256 / hmac_sha256
    sign_case: str                    # 签名大小写：upper / lower
    timestamp_unit: str               # 时间戳单位：ms(毫秒) / s(秒)
    timestamp_location: str           # 时间戳位置：query / header / body
    sign_location: str                # 签名位置：query / header / body
    sign_fields: List[str]            # 参与签名的字段名列表（按此顺序）
    sign_prefix: str                  # 签名字符串前缀
    sign_suffix: str                  # 签名字符串后缀
    secret_position: str              # 密钥位置：append / prefix / surround
    join_separator: str               # 字段拼接分隔符
    kv_connector: str                 # key-value 连接符
    url_match_pattern: Optional[str]  # URL 匹配正则（为空则匹配所有）
    created_at: str                   # 创建时间 ISO 格式
    updated_at: str                   # 更新时间 ISO 格式
```

### 3.3 模块设计

#### 模块一：持久化与数据服务层（services/signature_config_service.py）

新增服务模块，职责：
- 签名配置的增删改查（CRUD）
- JSON 文件的持久化读写（原子写：tmp + replace）
- 配置列表缓存（可选，避免频繁读文件）
- 获取所有启用的配置，供执行层使用

```python
"""签名配置数据服务。

职责：
- 签名配置的 CRUD 操作
- JSON 文件持久化（原子写）
- 提供执行层可调用的配置查询接口

开发导读:
- 存储位置: uploaded_collections/signature_configs.json
- 所有操作均为同步阻塞，保持与现有服务层一致
- app_secret 以明文存储在本地 JSON 中，不涉及网络传输
"""

import json
import uuid
from datetime import datetime
from pathlib import Path
from typing import Any, Dict, List, Optional

from postman_api_tester.report_server_app import ReportServerApp


_SIGNATURE_CONFIGS_PATH: Path = Path("uploaded_collections/signature_configs.json")


def configure_signature_configs_path(path: Path) -> None:
    global _SIGNATURE_CONFIGS_PATH
    _SIGNATURE_CONFIGS_PATH = path


def _load_configs() -> List[Dict[str, Any]]:
    if not _SIGNATURE_CONFIGS_PATH.exists():
        return []
    try:
        with _SIGNATURE_CONFIGS_PATH.open("r", encoding="utf-8") as f:
            data = json.load(f)
        if isinstance(data, list):
            return data
    except Exception:
        pass
    return []


def _save_configs(configs: List[Dict[str, Any]]) -> None:
    _SIGNATURE_CONFIGS_PATH.parent.mkdir(parents=True, exist_ok=True)
    tmp_path = _SIGNATURE_CONFIGS_PATH.with_suffix(".tmp")
    with tmp_path.open("w", encoding="utf-8") as f:
        json.dump(configs, f, indent=2, ensure_ascii=False)
    tmp_path.replace(_SIGNATURE_CONFIGS_PATH)


def list_signature_configs(enabled_only: bool = False) -> List[Dict[str, Any]]:
    configs = _load_configs()
    if enabled_only:
        configs = [c for c in configs if c.get("enabled", False)]
    return configs


def get_signature_config(config_id: str) -> Optional[Dict[str, Any]]:
    for cfg in _load_configs():
        if str(cfg.get("id", "")) == config_id:
            return cfg
    return None


def create_signature_config(config_data: Dict[str, Any]) -> Dict[str, Any]:
    configs = _load_configs()
    now = datetime.now().isoformat()
    new_config = {
        "id": uuid.uuid4().hex,
        "enabled": bool(config_data.get("enabled", True)),
        "name": str(config_data.get("name", "")).strip() or "未命名配置",
        "app_key": str(config_data.get("app_key", "")).strip(),
        "app_secret": str(config_data.get("app_secret", "")).strip(),
        "app_key_param": str(config_data.get("app_key_param", "app_key")).strip() or "app_key",
        "timestamp_param": str(config_data.get("timestamp_param", "timestamp")).strip() or "timestamp",
        "sign_param": str(config_data.get("sign_param", "sign")).strip() or "sign",
        "sign_algorithm": str(config_data.get("sign_algorithm", "md5")).strip().lower() or "md5",
        "sign_case": str(config_data.get("sign_case", "upper")).strip().lower() or "upper",
        "timestamp_unit": str(config_data.get("timestamp_unit", "ms")).strip().lower() or "ms",
        "timestamp_location": str(config_data.get("timestamp_location", "query")).strip().lower() or "query",
        "sign_location": str(config_data.get("sign_location", "query")).strip().lower() or "query",
        "sign_fields": list(config_data.get("sign_fields", []) or []),
        "sign_prefix": str(config_data.get("sign_prefix", "")).strip(),
        "sign_suffix": str(config_data.get("sign_suffix", "")).strip(),
        "secret_position": str(config_data.get("secret_position", "append")).strip().lower() or "append",
        "join_separator": str(config_data.get("join_separator", "")).strip(),
        "kv_connector": str(config_data.get("kv_connector", "")).strip(),
        "url_match_pattern": config_data.get("url_match_pattern") or None,
        "created_at": now,
        "updated_at": now,
    }
    configs.append(new_config)
    _save_configs(configs)
    return new_config


def update_signature_config(config_id: str, config_data: Dict[str, Any]) -> Optional[Dict[str, Any]]:
    configs = _load_configs()
    for idx, cfg in enumerate(configs):
        if str(cfg.get("id", "")) == config_id:
            now = datetime.now().isoformat()
            for key, value in config_data.items():
                if key in cfg and key not in ("id", "created_at"):
                    cfg[key] = value
            cfg["updated_at"] = now
            configs[idx] = cfg
            _save_configs(configs)
            return cfg
    return None


def delete_signature_config(config_id: str) -> bool:
    configs = _load_configs()
    original_len = len(configs)
    configs = [c for c in configs if str(c.get("id", "")) != config_id]
    if len(configs) < original_len:
        _save_configs(configs)
        return True
    return False


def find_matching_signature_config(url: str) -> Optional[Dict[str, Any]]:
    """根据 URL 查找第一个匹配的启用配置。"""
    import re
    for cfg in list_signature_configs(enabled_only=True):
        pattern = cfg.get("url_match_pattern")
        if not pattern:
            return cfg
        if re.search(str(pattern), str(url)):
            return cfg
    return None
```

新增 handler 模块，提供签名配置的 RESTful API：

```python
"""签名配置管理路由处理函数。

职责：
- 签名配置的增删改查接口
- 返回前端可直接消费的 JSON 响应
"""

from typing import Any, Dict

from flask import jsonify, request
from flask.typing import ResponseReturnValue

from postman_api_tester.handlers.base_handler import BaseHandler
from postman_api_tester.services.signature_config_service import (
    create_signature_config,
    delete_signature_config,
    get_signature_config,
    list_signature_configs,
    update_signature_config,
)


def _json_error(message: str, status_code: int) -> ResponseReturnValue:
    from postman_api_tester.exceptions import ValidationError
    return BaseHandler.error_response(ValidationError(message), status_code)


def api_signature_configs() -> ResponseReturnValue:
    """GET /api/signature-configs"""
    configs = list_signature_configs()
    # 响应中脱敏 app_secret
    safe_configs = []
    for cfg in configs:
        safe_cfg = dict(cfg)
        if safe_cfg.get("app_secret"):
            safe_cfg["app_secret"] = "***"
        safe_configs.append(safe_cfg)
    return jsonify({"code": 200, "message": "OK", "data": safe_configs})


def api_signature_config_create() -> ResponseReturnValue:
    """POST /api/signature-configs"""
    payload = request.get_json(silent=True) or {}
    if not isinstance(payload, dict):
        return _json_error("请求体必须是 JSON 对象", 400)
    name = str(payload.get("name", "")).strip()
    if not name:
        return _json_error("配置名称不能为空", 400)
    app_key = str(payload.get("app_key", "")).strip()
    app_secret = str(payload.get("app_secret", "")).strip()
    if not app_key or not app_secret:
        return _json_error("app_key 和 app_secret 不能为空", 400)

    config = create_signature_config(payload)
    # 响应中脱敏
    safe_config = dict(config)
    safe_config["app_secret"] = "***"
    return jsonify({"code": 200, "message": "创建成功", "data": safe_config})


def api_signature_config_update(config_id: str) -> ResponseReturnValue:
    """PUT /api/signature-configs/<config_id>"""
    payload = request.get_json(silent=True) or {}
    if not isinstance(payload, dict):
        return _json_error("请求体必须是 JSON 对象", 400)

    updated = update_signature_config(config_id, payload)
    if not updated:
        return _json_error("配置不存在", 404)

    safe_config = dict(updated)
    safe_config["app_secret"] = "***"
    return jsonify({"code": 200, "message": "更新成功", "data": safe_config})


def api_signature_config_delete(config_id: str) -> ResponseReturnValue:
    """DELETE /api/signature-configs/<config_id>"""
    success = delete_signature_config(config_id)
    if not success:
        return _json_error("配置不存在", 404)
    return jsonify({"code": 200, "message": "删除成功", "data": None})
```

#### 模块五：执行层集成（executor.py）

修改 `PostmanTestExecutor.execute_test()`，在请求发送前插入预处理：

```python
def execute_test(self) -> TestResultRecord:
    # ... 原有逻辑：构建请求参数 ...
    
    # 新增：请求签名预处理
    try:
        from postman_api_tester.services.signature_config_service import find_matching_signature_config
        from postman_api_tester.utils.request_preprocessor import (
            apply_request_signature,
            should_apply_signature,
        )
        
        url = self.api_config.get("url", "")
        if isinstance(url, dict):
            url = url.get("raw", "")
        
        sign_config = find_matching_signature_config(str(url))
        if sign_config and should_apply_signature(self.api_config, sign_config):
            apply_request_signature(self.api_config, sign_config)
            logger.debug("已应用签名配置: %s", sign_config.get("name", ""))
    except Exception as exc:
        logger.warning("请求签名预处理失败，继续使用原始参数: %s", exc)
    
    # ... 原有发送逻辑 ...
```

**关键变更**：
- 不再从 `config.py` 读取单一签名配置
- 改为从 `signature_config_service` 动态查询匹配的启用配置
- 支持多套配置并存，按 URL 正则匹配
- 预处理失败时不阻断执行

### 3.4 调用链路变更

```
PostmanTestExecutor.execute_test()
  ├─ 原有：从 api_config 读取参数
  ├─ 【新增】请求签名预处理（条件触发）
  │     ├─ 查询所有启用的签名配置（从 signature_configs.json）
  │     ├─ 按 URL 正则匹配第一个适用配置
  │     ├─ 生成最新时间戳
  │     ├─ 注入 app_key + timestamp
  │     ├─ 按规则计算 sign
  │     └─ 注入 sign
  ├─ 原有：request_builder 构建 requests 参数
  └─ 原有：session.get/post() 发送请求
```

### 3.5 边界情况处理

| 场景 | 处理策略 |
|------|---------|
| 无签名配置或全部禁用 | 跳过预处理，正常执行 |
| URL 不匹配任何配置的规则 | 跳过预处理 |
| 匹配到多个配置 | 取第一个匹配的配置（按列表顺序） |
| 签名配置不完整（缺少 app_key/app_secret） | 跳过预处理，记录 warning |
| 签名计算异常 | 捕获异常，跳过预处理，记录 warning，不阻断执行 |
| app_secret 在 Web 响应中泄露 | 所有 API 返回均脱敏为 `***` |
| 与现有 Token 认证并存 | 签名预处理在 Token 设置之后执行，两者独立不冲突 |
| 重试场景 | 每次重试都会重新触发预处理，生成新的时间戳和签名 |
| 多 Collection 共享同一配置 | 同一签名配置可被多个 Collection 复用 |
| 同一 Collection 不同接口使用不同配置 | 通过 url_match_pattern 区分匹配 |

## 六、待确认信息

由于 APIFox 分享链接为私有链接，无法直接访问接口文档，以下关键信息需要用户提供：

### 6.1 必选项（决定能否实施签名计算逻辑）

1. **签名算法**：MD5 / SHA256 / HMAC-SHA256 / 其他？
2. **时间戳单位**：秒(s) / 毫秒(ms)？
3. **时间戳参数名**：`timestamp` / `t` / 其他？
4. **签名参数名**：`sign` / `signature` / 其他？
5. **应用标识参数名**：`app_key` / `app_id` / 其他？
6. **app_key 和 app_secret 值**

### 6.2 可选项（影响签名计算规则）

7. **参与签名的字段列表及顺序**：如 `["app_key", "timestamp"]`
8. **字段拼接方式**：key-value 连接符、字段间分隔符
9. **密钥拼接位置**：末尾追加 / 开头 / 前后包裹
10. **签名大小写**：大写 / 小写
11. **签名位置**：query / header / body
12. **是否有 sign_prefix / sign_suffix**
13. **URL 匹配规则**：是否需要仅对特定 URL 模式签名

### 6.3 最快捷的获取方式

- **APIFox 前置脚本**：在 APIFox 中打开该接口，查看 Pre-request Script，贴出完整代码
- **抓包数据**：用 APIFox 成功请求一次，截图或导出实际发送的请求参数

## 八、风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 签名规则获取不全导致计算错误 | 高 | 用户提供 APIFox 前置脚本或抓包数据 |

| 与现有请求链路兼容性 | 低 | 预处理失败降级为原始参数，不阻断执行 |
