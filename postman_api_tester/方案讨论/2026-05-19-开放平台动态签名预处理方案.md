# 开放平台动态签名预处理方案

版本：方案讨论版
生成日期：2026-05-19
文档定位：解决从 APIFox 导出 Postman JSON 后执行报错"v1版本标准开放-时间戳异常(超时或错误)"的问题，设计请求发送前动态签名注入机制。

---

## 一、问题现象与根因分析

### 1.1 现象描述

用户从 APIFox 导出 Postman Collection JSON 文件后，在本项目执行测试，接口返回：

```json
{
  "message": "v1版本标准开放-时间戳异常(超时或错误)"
}
```

### 1.2 根因分析

该报错是典型的**开放平台 API 签名时间戳校验失败**，核心原因如下：

| 环节 | 问题 |
|------|------|
| APIFox 调试时 | 时间戳和签名由 APIFox 的**前置脚本**或**动态变量**实时生成，每次请求都是最新的 |
| 导出 Postman JSON 时 | 时间戳和签名被**静态固化**到 JSON 文件中，成为固定值 |
| 本项目执行时 | 直接读取 JSON 中的静态参数发送请求，时间戳已过期 |
| 服务端校验时 | 发现时间戳超出有效窗口（通常 ±30s~300s），返回时间戳异常 |

**本质**：本项目当前仅支持"静态请求配置 -> 直接发送"的执行模式，缺少"请求发送前动态预处理"的能力。

### 1.3 开放平台签名机制通用特征

根据行业通用实践，开放平台签名通常包含以下要素：

1. **时间戳参数**：`timestamp`，单位通常为秒(s)或毫秒(ms)
2. **应用标识**：`app_key` / `app_id` / `access_key`
3. **签名参数**：`sign` / `signature`
4. **签名算法**：MD5 / SHA256 / HMAC-SHA256 等
5. **签名字段排序**：按参数名 ASCII 升序排序后拼接
6. **密钥拼接**：末尾拼接 `app_secret`
7. **可选随机数**：`nonce` 防重放攻击

## 三、方案设计

### 3.1 总体思路

新增**请求签名预处理器（Request Signature Preprocessor）**，在 `PostmanTestExecutor.execute_test()` 发送请求前调用，根据配置规则：

1. 识别当前接口是否需要签名（通过 URL 匹配或配置开关）
2. 动态生成最新时间戳
3. 按签名规则重新计算签名值
4. 将更新后的时间戳和签名注入到请求参数中
5. 继续原有请求发送流程

#### 模块二：签名预处理工具（utils/request_preprocessor.py）

新增纯工具模块，职责单一：

```python
"""请求签名预处理器。

职责：
- 根据配置规则，在请求发送前动态更新时间戳和重新计算签名。
- 支持多种签名算法（MD5/SHA256/HMAC-SHA256）和字段排序规则。
- 纯函数工具，不持有业务状态。

入口：apply_request_signature(api_config, sign_config)
"""

import hashlib
import hmac
import time
from typing import Any, Dict, List, Optional


def _generate_timestamp(unit: str = "ms") -> int:
    """生成当前时间戳。"""
    if unit == "s":
        return int(time.time())
    return int(time.time() * 1000)


def _calculate_sign_md5(sign_string: str, case: str = "upper") -> str:
    """计算 MD5 签名。"""
    result = hashlib.md5(sign_string.encode("utf-8")).hexdigest()
    return result.upper() if case == "upper" else result.lower()


def _calculate_sign_sha256(sign_string: str, case: str = "upper") -> str:
    """计算 SHA256 签名。"""
    result = hashlib.sha256(sign_string.encode("utf-8")).hexdigest()
    return result.upper() if case == "upper" else result.lower()


def _calculate_sign_hmac_sha256(sign_string: str, secret: str, case: str = "upper") -> str:
    """计算 HMAC-SHA256 签名。"""
    result = hmac.new(
        secret.encode("utf-8"),
        sign_string.encode("utf-8"),
        hashlib.sha256,
    ).hexdigest()
    return result.upper() if case == "upper" else result.lower()


def _build_sign_string(
    params: Dict[str, Any],
    sign_fields: List[str],
    app_secret: str,
    join_separator: str = "",
    kv_connector: str = "",
    secret_position: str = "append",
    sign_prefix: str = "",
    sign_suffix: str = "",
) -> str:
    """按规则构建签名字符串。"""
    # 按 sign_fields 顺序提取字段值
    field_values = []
    for field in sign_fields:
        value = params.get(field, "")
        field_values.append(f"{field}{kv_connector}{value}")
    
    sign_string = join_separator.join(field_values)
    
    # 根据密钥位置拼接
    if secret_position == "append":
        sign_string = sign_string + app_secret
    elif secret_position == "prefix":
        sign_string = app_secret + sign_string
    elif secret_position == "surround":
        sign_string = app_secret + sign_string + app_secret
    
    if sign_prefix:
        sign_string = sign_prefix + sign_string
    if sign_suffix:
        sign_string = sign_string + sign_suffix
    
    return sign_string


def _inject_param(
    api_config: Dict[str, Any],
    param_name: str,
    param_value: Any,
    location: str,
) -> None:
    """将参数注入到请求指定位置。"""
    if location == "query":
        # 注入到 URL query 参数
        url_obj = api_config.get("url", "")
        if isinstance(url_obj, dict):
            query_list = url_obj.get("query", [])
            if isinstance(query_list, list):
                # 移除同名旧参数
                url_obj["query"] = [
                    q for q in query_list
                    if str(q.get("key", "")) != param_name
                ]
                url_obj["query"].append({"key": param_name, "value": str(param_value)})
        # 同时更新 params
        params = api_config.get("params", {})
        if isinstance(params, dict):
            params[param_name] = param_value
    elif location == "header":
        # 注入到 Headers
        headers = api_config.get("headers", {})
        if isinstance(headers, dict):
            headers[param_name] = str(param_value)
    elif location == "body":
        # 注入到 Body（仅支持 JSON body）
        body = api_config.get("body", {})
        if isinstance(body, dict):
            body[param_name] = param_value


def apply_request_signature(
    api_config: Dict[str, Any],
    sign_config: Dict[str, Any],
) -> bool:
    """对单个接口配置应用动态签名。
    
    Args:
        api_config: 接口配置（会被原地修改）
        sign_config: 签名配置
    
    Returns:
        是否成功应用签名
    """
    if not sign_config.get("app_key") or not sign_config.get("app_secret"):
        return False
    
    # 1. 生成最新时间戳
    timestamp = _generate_timestamp(sign_config.get("timestamp_unit", "ms"))
    timestamp_param = sign_config.get("timestamp_param", "timestamp")
    timestamp_location = sign_config.get("timestamp_location", "query")
    
    # 2. 注入 app_key
    app_key_param = sign_config.get("app_key_param", "app_key")
    app_key_location = sign_config.get("app_key_location", timestamp_location)
    _inject_param(api_config, app_key_param, sign_config["app_key"], app_key_location)
    
    # 3. 注入时间戳
    _inject_param(api_config, timestamp_param, str(timestamp), timestamp_location)
    
    # 4. 收集参与签名的参数
    sign_fields = sign_config.get("sign_fields", [])
    sign_params: Dict[str, Any] = {}
    for field in sign_fields:
        # 从 query/headers/body 中提取字段值
        # 简化处理：优先从 api_config.params 读取
        params = api_config.get("params", {})
        if isinstance(params, dict) and field in params:
            sign_params[field] = params[field]
        else:
            sign_params[field] = ""
    
    # 5. 构建签名字符串并计算签名
    sign_string = _build_sign_string(
        sign_params,
        sign_fields,
        sign_config["app_secret"],
        join_separator=sign_config.get("join_separator", ""),
        kv_connector=sign_config.get("kv_connector", ""),
        secret_position=sign_config.get("secret_position", "append"),
        sign_prefix=sign_config.get("sign_prefix", ""),
        sign_suffix=sign_config.get("sign_suffix", ""),
    )
    
    algorithm = sign_config.get("sign_algorithm", "md5")
    case = sign_config.get("sign_case", "upper")
    if algorithm == "md5":
        sign_value = _calculate_sign_md5(sign_string, case)
    elif algorithm == "sha256":
        sign_value = _calculate_sign_sha256(sign_string, case)
    elif algorithm == "hmac_sha256":
        sign_value = _calculate_sign_hmac_sha256(sign_string, sign_config["app_secret"], case)
    else:
        sign_value = _calculate_sign_md5(sign_string, case)
    
    # 6. 注入签名
    sign_param = sign_config.get("sign_param", "sign")
    sign_location = sign_config.get("sign_location", "query")
    _inject_param(api_config, sign_param, sign_value, sign_location)
    
    return True


def should_apply_signature(
    api_config: Dict[str, Any],
    sign_config: Dict[str, Any],
) -> bool:
    """判断当前接口是否需要应用签名。"""
    url_pattern = sign_config.get("url_match_pattern")
    if not url_pattern:
        return True
    
    url = api_config.get("url", "")
    if isinstance(url, dict):
        url = url.get("raw", "")
    import re
    return bool(re.search(url_pattern, str(url)))
```

#### 模块三：执行层集成（executor.py）

在 `PostmanTestExecutor.execute_test()` 中，请求构建后、发送前，插入预处理调用：

```python
def execute_test(self) -> TestResultRecord:
    # ... 原有逻辑 ...
    
    # 新增：请求签名预处理
    try:
        from postman_api_tester import config as _cfg
        if getattr(_cfg, "OPEN_API_SIGN_ENABLED", False):
            sign_config = getattr(_cfg, "OPEN_API_SIGN_CONFIG", {})
            from postman_api_tester.utils.request_preprocessor import (
                apply_request_signature,
                should_apply_signature,
            )
            if should_apply_signature(self.api_config, sign_config):
                apply_request_signature(self.api_config, sign_config)
    except Exception as exc:
        logger.warning("请求签名预处理失败，继续使用原始参数: %s", exc)
    
    # ... 原有发送逻辑 ...
```

**注意**：预处理失败时不阻断执行，降级为原始参数（避免签名配置错误导致全部接口无法执行）。

### 3.3 调用链路变更

```
PostmanTestExecutor.execute_test()
  ├─ 原有：从 api_config 读取参数
  ├─ 【新增】请求签名预处理（条件触发）
  │     ├─ 检查 OPEN_API_SIGN_ENABLED
  │     ├─ 匹配 URL 规则
  │     ├─ 生成最新时间戳
  │     ├─ 注入 app_key + timestamp
  │     ├─ 按规则计算 sign
  │     └─ 注入 sign
  ├─ 原有：request_builder 构建 requests 参数
  └─ 原有：session.get/post() 发送请求
```

### 3.4 边界情况处理

| 场景 | 处理策略 |
|------|---------|
| 签名配置不完整（缺少 app_key/app_secret） | 跳过预处理，记录 warning |
| 签名计算异常 | 捕获异常，跳过预处理，记录 warning，不阻断执行 |
| URL 不匹配配置的正则 | 跳过预处理 |
| 时间戳单位配置错误 | 默认使用毫秒(ms) |
| 签名算法配置不支持 | 默认使用 MD5 |
| 与现有 Token 认证并存 | 签名预处理在 Token 设置之后执行，两者独立不冲突 |
| 重试场景 | 每次重试都会重新触发预处理，生成新的时间戳和签名 |

## 六、替代方案（快速验证）

如果上述签名规则获取困难，可先尝试以下**最小成本验证**：

### 方案 C：Postman 预请求脚本

在 Postman 客户端中编辑 Collection，添加 Pre-request Script 动态计算签名，然后导出 JSON。但 Postman 的预请求脚本不会被本项目执行（本项目不解析 Postman 脚本）。

### 方案 D（推荐）：按本文方案实施

在本项目中增加签名预处理能力，一次性解决所有类似开放平台接口的测试问题。
