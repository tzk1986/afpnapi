## 改进一：Headers/Params 批量文本编辑模式

### 1.1 现状分析

当前 `renderKVTab()` 仅支持逐行 KV 表格编辑（`collection_editor.html:1343-1356`）：

```javascript
function renderKVTab(kvs, field) {
    // 仅渲染 <input Key> <input Value> <button 删除> 的逐行表格
    // + "添加" 按钮
}
```

**痛点**：从抓包工具、文档或 Postman 中复制 Headers 时，通常是 `Key: Value` 多行文本格式，需逐行手动拆分录入。

### 1.2 竞品参考

- **Postman**：Headers/Params 面板有 "Bulk Edit" 按钮，切换到文本模式，格式 `key: value`
- **Insomnia**：同样支持批量文本编辑，`#` 开头的行自动禁用
- **Bruno**：支持 raw text 模式编辑 Headers

### 1.3 技术方案

**修改文件**：`templates/collection_editor.html`

#### 1.3.1 UI 变更

在 `renderKVTab()` 中增加模式切换按钮：

```
┌──────────────────────────────────────────┐
│ [表单编辑] [批量编辑]            + 添加   │
├──────────────────────────────────────────┤
│ Key1: Value1                             │
│ Key2: Value2                             │
│ # DisabledKey: DisabledValue             │
│                                          │
└──────────────────────────────────────────┘
```

#### 1.3.2 核心逻辑

```javascript
// state 新增字段
state.kvEditMode = { headers: 'form', params: 'form' };

// 切换编辑模式
function switchKVEditMode(field, mode) {
    state.kvEditMode[field] = mode;
    if (mode === 'bulk') {
        // KV 数组 → 文本
        const text = (state.editBuffer[field] || [])
            .map(kv => kv.disabled ? `# ${kv.key}: ${kv.value}` : `${kv.key}: ${kv.value}`)
            .join('\n');
        state._bulkText = { [field]: text };
    } else {
        // 文本 → KV 数组
        const text = state._bulkText?.[field] || '';
        state.editBuffer[field] = parseBulkKV(text);
        syncToAllRequests();
    }
    renderEditor();
}

// 解析批量文本
function parseBulkKV(text) {
    return text.split('\n')
        .filter(line => line.trim())
        .map(line => {
            const disabled = line.trimStart().startsWith('#');
            const clean = disabled ? line.trimStart().slice(1).trim() : line;
            const colonIdx = clean.indexOf(':');
            if (colonIdx < 0) return { key: clean.trim(), value: '', disabled };
            return {
                key: clean.slice(0, colonIdx).trim(),
                value: clean.slice(colonIdx + 1).trim(),
                disabled
            };
        });
}
```

#### 1.3.3 KV 数据结构扩展

现有 KV 结构 `{key, value}` 扩展为 `{key, value, disabled}`（向后兼容，`disabled` 字段可选）：

- `disabled: true` → 该 Header/Param 不参与请求发送
- 后端 `executor.py` 中 headers/params 注入时过滤 `disabled` 项
- Parser 解析时忽略 `disabled` 字段（向后兼容旧 Collection JSON）

#### 1.3.4 影响范围

| 文件 | 改动 |
|------|------|
| `templates/collection_editor.html` | `renderKVTab()` 增加模式切换、`parseBulkKV()`、`switchKVEditMode()` |
| `postman_api_tester/executor.py` | headers/params 注入时过滤 disabled 项（约 2 行） |
| `postman_api_tester/parser.py` | 解析 headers/params 时保留 disabled 字段（约 1 行） |

## 改进二：JSONPath 响应过滤器

### 2.1 现状分析

报告详情页的响应体查看器（`report_view.html:2803-2814`）目前仅支持：
- 展开/收起 JSON 树
- 复制 JSON 文本
- 预览/完整模式切换

**痛点**：大型 JSON 响应（如列表接口返回数百条记录）难以快速定位目标字段。用户需复制全文到外部工具再用 JSONPath 查询。

### 2.2 竞品参考

- **Postman**：响应区支持 JSONPath 搜索，高亮匹配节点
- **Insomnia**：Filter 输入框 + 实时过滤 JSON 树
- **Hoppscotch**：JSONPath 表达式实时预览

### 2.3 技术方案

**修改文件**：`templates/report_view.html`

#### 2.3.1 UI 变更

在响应体工具栏（展开/收起/复制按钮旁）增加 JSONPath 输入框：

```
┌─ 响应体 ────────────────────────────────────────────┐
│ [展开] [收起] [复制]  │ $.data[*].name  [过滤] [清除] │
├──────────────────────────────────────────────────────┤
│ ▼ {                                                  │
│   "data": [                                          │
│     { "name": "张三", ... }     ← 高亮匹配            │
│     { "name": "李四", ... }     ← 高亮匹配            │
│   ]                                                  │
│ }                                                    │
└──────────────────────────────────────────────────────┘
```

#### 2.3.2 核心逻辑

采用纯前端 JS 实现 JSONPath 过滤（无需后端 API）：

```javascript
// 轻量 JSONPath 解析器（支持常用语法）
function filterJsonByPath(data, path) {
    // 支持的语法：
    // $           - 根节点
    // .key        - 对象属性
    // [*]         - 数组所有元素
    // [n]         - 数组索引
    // ..key       - 递归查找（descendant）
    // 返回匹配结果数组
}

function applyJsonPathFilter() {
    const path = document.getElementById('jsonPathInput').value.trim();
    if (!path) {
        // 清空过滤，恢复原始 JSON 树
        renderResponseBody(currentDetailData.response_info.body, ...);
        return;
    }
    const body = currentDetailData.response_info.body;
    const parsed = typeof body === 'string' ? JSON.parse(body) : body;
    const results = filterJsonByPath(parsed, path);
    // 渲染过滤后的结果
    renderJsonViewer('responseBodyView', results, { collapsedDepth: 2 });
}
```

#### 2.3.3 JSONPath 语法支持范围

| 语法 | 示例 | 说明 |
|------|------|------|
| `$` | `$` | 根对象 |
| `.key` | `$.data.name` | 属性访问 |
| `[*]` | `$.data[*]` | 数组通配 |
| `[n]` | `$.data[0]` | 数组索引 |
| `..key` | `$..id` | 递归查找 |
| `[?(@.key)]` | `$..[?(@.status)]` | 存在性过滤（可选） |

**不实现**：`[?(@.price>10)]` 表达式过滤（复杂度高，使用场景少）。

#### 2.3.4 影响范围

| 文件 | 改动 |
|------|------|
| `templates/report_view.html` | 响应体工具栏增加 JSONPath 输入框、`filterJsonByPath()`、`applyJsonPathFilter()` |
| `templates/collection_editor.html` | 编辑器响应区同步增加（可选，复用同一函数） |

## 改进三：响应体文本搜索与高亮

### 3.1 现状分析

报告详情页和执行结果弹窗中的 JSON 响应树不支持文本搜索。当 JSON 层级较深（5+ 层嵌套）时，用户需逐层展开查找目标值。

### 3.2 竞品参考

- **Postman**：响应区 Ctrl+F 搜索，匹配项高亮 + 计数
- **Chrome DevTools**：Network 面板响应搜索，支持正则
- **Bruno**：响应体搜索框 + 上下导航

### 3.3 技术方案

**修改文件**：`templates/report_view.html`

#### 3.3.1 UI 变更

在响应体卡片头部增加搜索按钮（🔍 图标），点击展开搜索栏：

```
┌─ 响应体 ────────────────────────────────────────────┐
│ [展开] [收起] [复制] [🔍]  │ 🔍 token  [◀] [▶] 3/12 │
├──────────────────────────────────────────────────────┤
│ ▼ {                                                  │
│   "access_token": "eyJhb..."  ← 高亮 "token"         │
│   "data": {                                          │
│     "token_type": "Bearer"    ← 高亮 "token"         │
│   }                                                  │
│ }                                                    │
└──────────────────────────────────────────────────────┘
```

#### 3.3.2 核心逻辑

```javascript
function searchInJsonViewer(keyword) {
    const viewer = document.getElementById('responseBodyView');
    // 1. 清除之前的高亮
    viewer.querySelectorAll('.search-highlight').forEach(el => {
        el.outerHTML = el.textContent; // 还原
    });
    // 2. 在 JSON 树的文本节点中查找 keyword
    const walker = document.createTreeWalker(viewer, NodeFilter.SHOW_TEXT);
    const matches = [];
    while (walker.nextNode()) {
        const node = walker.currentNode;
        if (node.textContent.toLowerCase().includes(keyword.toLowerCase())) {
            // 3. 包装匹配文本为 <mark class="search-highlight">
            highlightTextNode(node, keyword, matches);
            // 4. 展开包含匹配项的父级 <details>
            expandParentDetails(node);
        }
    }
    // 5. 更新计数并滚动到第一个匹配
    state.searchMatches = matches;
    state.searchIndex = 0;
    navigateSearchMatch(0);
}
```

#### 3.3.3 快捷键支持

- `Ctrl+F`（在响应体区域）：激活搜索框
- `Enter` / `F3`：下一个匹配
- `Shift+Enter` / `Shift+F3`：上一个匹配
- `Escape`：关闭搜索

#### 3.3.4 影响范围

| 文件 | 改动 |
|------|------|
| `templates/report_view.html` | 搜索栏 UI、`searchInJsonViewer()`、高亮/导航逻辑、快捷键绑定 |
| CSS | `.search-highlight`（黄色背景）、`.search-highlight.current`（橙色边框） |
