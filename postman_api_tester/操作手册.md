## 🚀 快速开始

### 1️⃣ 准备 Postman 导出文件

从 **APIFox** 或 **Postman** 导出接口集合：

```
步骤：
1. 打开 APIFox 或 Postman
2. 右键点击集合 → 导出
3. 选择 Postman (Collection v2.1) 格式
4. 保存为 JSON 文件（例如：api.json）
```

**导出文件示例结构：**
```json
{
  "info": {
    "name": "我的API集合",
    "schema": "https://schema.getpostman.com/json/collection/v2.1.0/collection.json"
  },
  "variable": [
    { "key": "baseUrl", "value": "https://api.example.com" }
  ],
  "item": [
    {
      "name": "获取用户",
      "request": {
        "method": "GET",
        "url": "https://api.example.com/users",
        "header": [
          { "key": "Accept", "value": "application/json" }
        ]
      }
    }
  ]
}
```

### 2️⃣ 运行测试

**方式 A - 从包目录运行：**
```bash
cd postman_api_tester
python postman_api_tester.py ../api.json
```

**方式 B - 从根目录运行：**
```bash
python postman_api_tester/postman_api_tester.py api.json
```

**方式 C - 指定基础 URL：**
```bash
python postman_api_tester/postman_api_tester.py api.json https://api.example.com
```

**方式 D - 指定报告输出目录：**
```bash
python postman_api_tester/postman_api_tester.py api.json https://api.example.com ./my_reports
```

### 3️⃣ 查看报告

- **HTML 报告**：`reports/postman_report_YYYYMMDD_HHMMSS.html`
- **控制台输出**：运行完成后立即显示

## ⚠️ 遇到的问题和解决方案

### 问题1：Seldom TestCase 属性冲突

**现象：**
```
AttributeError: can't set attribute 'response'
AttributeError: can't set attribute 'status_code'
```

**原因：**
- `PostmanTestExecutor` 继承 `seldom.TestCase` 后
- `response` 和 `status_code` 被 TestCase 占用为只读属性
- 导致在 `__init__` 中无法初始化这些属性

**解决方案：**
❌ 不继承 `seldom.TestCase`
✅ 直接使用 `requests.Session` 进行 HTTP 操作

**代码改动：**
```python
# 之前（有问题）
class PostmanTestExecutor(seldom.TestCase):
    def __init__(self, api_config: Dict):
        super().__init__()
        self.response = None  # ❌ 属性冲突
        self.status_code = None  # ❌ 属性冲突

# 现在（已修复）
class PostmanTestExecutor:
    def __init__(self, api_config: Dict):
        import requests
        self.http_response = None  # ✅ 改用 http_response
        self.resp_status_code = None  # ✅ 改用 resp_status_code
        self.session = requests.Session()  # ✅ 直接使用 requests
```

### 问题2：编码错误（Windows PowerShell）

**现象：**
```
UnicodeEncodeError: 'gbk' codec can't encode character '\u2713'
```

**原因：**
- Windows PowerShell 默认编码为 GBK
- 无法正确显示 Unicode 字符（✓ 等）

**解决方案：**
在运行命令时不使用 pipe 管道：
```bash
# ❌ 有问题的命令
python test_postman_api_tester.py 2>&1 | tee results.txt

# ✅ 正确的命令
python test_postman_api_tester.py
```

### 问题3：SSL 连接错误

**现象：**
```
SSLEOFError(8, 'EOF occurred in violation of protocol (_ssl.c:1007)')
```

**原因：**
- 网络连接不稳定
- SSL/TLS 握手被中断

**解决方案：**
```python
# 在 requests 中禁用 SSL 验证（仅用于测试）
response = session.get(url, headers=headers, verify=False, timeout=10)

# 或者设置重试机制
from urllib3.util.retry import Retry
from requests.adapters import HTTPAdapter

session = requests.Session()
retry = Retry(total=3, backoff_factor=0.5)
adapter = HTTPAdapter(max_retries=retry)
session.mount('https://', adapter)
```

### 问题4：文件路径问题

**现象：**
```
FileNotFoundError: 文件不存在: api.json
```

**原因：**
- 相对路径指向不正确
- 运行目录与文件位置不匹配

**解决方案：**
```bash
# 明确指定相对路径
python postman_api_tester.py ../api.json        # 从包目录
python postman_api_tester.py api.json          # 从根目录
python postman_api_tester.py ./api.json        # 当前目录

# 使用绝对路径
python postman_api_tester.py D:\path\to\api.json
```
