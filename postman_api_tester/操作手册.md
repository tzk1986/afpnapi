# Postman API 测试工具 - 操作手册（v1.0.2）

## 1. 工具目标

用于读取 Postman/APIFox 导出的 Collection JSON，批量执行接口请求，输出控制台摘要与 HTML 报告。

## 2. 目录说明

- 核心代码：postman_api_tester/postman_api_tester.py
- 运行入口：python -m postman_api_tester.postman_api_tester
- 配置文件：postman_api_tester/config.py
- 快速启动脚本：postman_api_tester/run_test_and_open.py
- 报告输出目录：可配置（默认 reports/）
- 文档目录：postman_api_tester/

## 3. 前置要求

1. Python 3.10+
2. 已安装 requirements.txt 依赖
3. 具备可访问接口环境（内网/测试环境）

## 4. 标准执行流程

1. 准备 Postman JSON（建议 Collection v2.1）。
2. 检查并按需设置 config.py：BASE_URL、TOKEN。
3. 执行测试命令。
4. 查看控制台汇总。
5. 打开 HTML 报告检查失败详情。
6. 必要时基于新 token 重跑。

## 5. 命令行参数

```bash
python -m postman_api_tester.postman_api_tester <postman_file> [base_url] [output_dir] [token] [results_per_page]
```

- postman_file：必填，Postman JSON 文件路径
- base_url：可选，覆盖文件内 baseUrl
- output_dir：可选，默认项目根目录下 reports/
- token：可选，传入后跳过自动登录
- results_per_page：可选，报告每页条数，默认 30

配置优先级补充：

1. 命令行 `output_dir`
2. `postman_api_tester/config.py` 中 `REPORT_OUTPUT_DIR`
3. 默认 `项目根目录/reports/`

## 6. 报告说明

每次执行会生成：

- postman_report_时间戳.html：主报告
- postman_report_时间戳_details.json：详情数据
- postman_report_时间戳_meta.json：历史索引与报告比对元数据

报告中可查看：

- 请求方法、URL、状态码
- 请求头/参数/请求体
- 响应头/响应体
- 汇总统计（通过/失败/错误）

## 7. 历史报告与差异对比（1.0.2）

### 7.1 现状判断

1. 优化前项目已经支持保留历史 HTML 报告文件，因为每次报告都使用时间戳命名。
2. 优化前项目不支持可视化历史报告列表与两份报告的结构化差异对比。
3. 优化前文档和脚本提到了 `report_server.py`，但仓库中实际不存在该文件，因此无法真正提供浏览器服务，更无法给同局域网开发访问。

### 7.2 优化后能力

1. 新增项目根目录服务端：`python report_server.py`
2. 报告首页支持查看历史报告列表。
3. 支持选择两份历史报告进行差异对比。
4. 对比内容包含：成功率变化、新增接口、移除接口、状态变化接口。
5. 服务默认监听 `0.0.0.0:5000`，同局域网开发可直接访问。

### 7.3 启动方式

```bash
cd d:\tangzk\py\seldom-api-testing
python report_server.py
```

启动后可使用：

1. 本机访问：`http://127.0.0.1:5000`
2. 局域网访问：`http://你的局域网IP:5000`

### 7.4 使用说明

1. 打开首页后可查看历史总报告列表，分页子报告页不会在首页展示。
2. 点击报告名称或“数据视图”可直接进入单份报告的数据浏览页。
3. 在单报告页可按接口名称或路径搜索，并支持结果状态筛选。
4. 单报告页支持分页显示，默认 20 条，最高 100 条。
5. 点击“详情”可查看请求头、查询参数、请求体、响应头、响应体。
6. 点击“原始HTML”仍可打开原始静态报告页面。
7. 在页面下方选择两份报告，点击“开始对比”。
8. 若要给开发同事访问，只需保证双方在同一局域网，且本机防火墙已放通 5000 端口。

### 7.5 防火墙说明

如果其他机器无法访问，但本机可以访问，请检查 Windows 防火墙是否允许 Python 或 5000 端口的入站访问。

推荐直接使用项目根目录脚本一键放通：

```powershell
cd d:\tangzk\py\seldom-api-testing
powershell -ExecutionPolicy Bypass -File .\allow_report_server_firewall.ps1
```

说明：

1. 脚本会自动申请管理员权限。
2. 默认放通 TCP 5000 端口，规则名为 `Postman Report Server 5000`。
3. 执行完成后，同局域网可通过 `http://你的局域网IP:5000` 访问报告中心。

如需手工执行，也可以使用下面命令：

```powershell
New-NetFirewallRule -DisplayName "Postman Report Server 5000" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 5000
```

## 8. 迁移验证建议（MySQL -> 国产数据库）

该工具适合做业务回归与接口可用性验证：

1. 验证接口状态码与主流程可用性
2. 验证关键响应字段是否完整
3. 对迁移前后同入参做结果对比

仅靠接口通过率不能覆盖全部迁移风险，建议补充：

1. 数据一致性抽样核对
2. SQL 兼容性专项验证
3. 并发与性能压测

## 9. 故障排查

- 未登录/401：优先确认 token 来源和优先级
- URL错误：检查 base_url 覆盖逻辑
- 接口超时：确认网络、防火墙、目标服务健康状态
- 报告为空：确认 JSON 是否包含 item/request 结构
- 局域网不可访问：检查 `python report_server.py` 是否已启动，以及防火墙是否放通 5000 端口
- 无法对比历史报告：确认 `reports/` 下同时存在 `postman_report_*.html` 与对应的 `postman_report_*_meta.json`

## 12. 打包给其他测试同学（分步）

### 12.1 打包内容

1. `postman_api_tester/`
2. `report_server.py`
3. `allow_report_server_firewall.ps1`
4. `requirements.txt`
5. 需要执行的 `*.postman.json`

### 12.2 安装步骤

1. 安装 Python 3.10+
2. 进入项目目录：`cd d:\tangzk\py\seldom-api-testing`
3. 安装依赖：`pip install -r requirements.txt`
4. 修改 `postman_api_tester/config.py`：`BASE_URL`、`TOKEN`、`REPORT_OUTPUT_DIR`

### 12.3 执行命令

1. 执行测试：`python -m postman_api_tester.postman_api_tester "支付（UPS）.postman.json"`
2. 启动报告中心：`python report_server.py`
3. 浏览器访问：`http://127.0.0.1:5000`

### 12.4 报告目录一致性

报告中心读取路径优先级：

1. `POSTMAN_REPORTS_DIR`
2. `REPORTS_DIR`
3. `postman_api_tester/config.py` 的 `REPORT_OUTPUT_DIR`
4. 默认 `项目根目录/reports/`

按此规则可确保测试输出目录与报告中心读取目录一致。
